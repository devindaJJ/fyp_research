================================================================================
                    URBAN TRAFFIC SYSTEM - PROJECT SUMMARY
================================================================================

PROJECT OVERVIEW
================================================================================
The Urban Traffic System is a comprehensive traffic management and violation 
detection platform designed for Sri Lanka. It combines real-time traffic 
analysis, vehicle tracking, parking management, and automated violation 
detection to improve urban traffic flow and enforce parking regulations.

This is a Final Year Research (FYP) project focusing on:
- Real-time traffic flow analysis
- Automatic Number Plate Recognition (ANPR)
- Vehicle violation detection
- Smart parking management
- Route optimization with traffic awareness


================================================================================
CORE ARCHITECTURE & COMPONENTS
================================================================================

1. BACKEND SERVICES (Flask REST API)
   Location: /backend/
   
   Key Files:
   - app.py: Main Flask application with REST endpoints
   - google_sheets.py: Integration with Google Sheets for data persistence
   - traffic_routes.py: Traffic-related API blueprints
   - config.py: Configuration management
   
   Primary Endpoints:
   - GET  /api/parking-data: Retrieve parking spot data
   - GET  /api/violations: Get traffic violation alerts (last 24 hours)
   - GET  /api/statistics: System-wide statistics
   
   Features:
   - Google Sheets integration for data storage
   - Real-time parking availability tracking
   - Violation detection and logging
   - System health statistics


2. CORE TRAFFIC ANALYSIS ENGINE (Python)
   Location: /src/core/
   
   Key Modules:
   
   a) traffic_analyzer.py
      - Analyzes traffic conditions for routes
      - Suggests alternative routes when congestion detected
      - Determines congestion levels (light/moderate/heavy)
      - Provides delay estimation
      - Rerouting recommendations
      
   b) route_optimizer.py
      - Finds optimal routes based on user preferences
      - Supports preference parameters:
        * avoid_tolls
        * avoid_highways
        * travel_preference (fastest/shortest/scenic)
      - Scores routes based on time, distance, and preferences
      
   c) location_service.py
      - Automatic current location detection
      - Address geocoding and reverse geocoding
      - Location validation and processing
      
   d) config_manager.py
      - Loads and manages system configuration
      - Handles location data and threshold settings


3. GOOGLE MAPS INTEGRATION
   Location: /src/api/
   
   Components:
   - google_maps_client.py: Main client for Google Maps API
   - geolocation_api.py: IP-based location detection
   
   Capabilities:
   - Real-time traffic data fetching
   - Multiple route alternatives (up to 5 routes)
   - Distance matrix calculations
   - Geocoding and reverse geocoding
   - Supports traffic models: best_guess, pessimistic, optimistic


4. DATA MODELS
   Location: /src/models/
   
   Key Classes:
   - Route: Single route with origin, destination, duration, distance
   - RouteAlternative: Alternative routes with traffic levels
   - TrafficLevel: Enum for congestion levels (LIGHT, MODERATE, HEAVY)
   - TrafficAnalysis: Comprehensive analysis result containing all route data
   - Location: Geographic location with address and coordinates
   
   ML Models:
   - best_model.h5: Trained LSTM model for traffic prediction
   - final_model.h5: Final traffic prediction model


5. FRONTEND DASHBOARD (React)
   Location: /frontend/traffic-dashboard/
   
   Components:
   - App.js: Main application shell
   - Dashboard.js: Primary dashboard view
   - TrafficAnalyzer.js: Route analysis interface
   - ParkingChart.js: Parking availability visualization
   - SystemHealth.js: System status monitoring
   - StatisticsCards.js: Key metrics display
   - RecentActivity.js: Live activity feed
   - violationAlerts.js: Violation alerts display
   
   Features:
   - Real-time traffic visualization
   - Route planning interface
   - Parking spot availability
   - Violation alert notifications
   - System health monitoring
   - Activity logs


6. ESP32 HARDWARE INTEGRATION
   Location: /esp32/
   
   Components:
   - main.ino: Core microcontroller logic
   - sensor.h: Parking sensor integration
   - parking_logic.h: Parking space detection logic
   - network.h: WiFi connectivity management
   
   Purpose:
   - Real-time parking space monitoring
   - Sensor data collection
   - Vehicle detection


7. UTILITIES & HELPERS
   Location: /src/utils/
   
   Modules:
   - logger.py: Structured logging setup
   - validators.py: Input validation functions
   - helpers.py: Common utility functions
   
   Location: /helper/
   - google_maps_api.py: Additional Google Maps utilities

================================================================================
MAIN WORKFLOW EXAMPLES
================================================================================

SCENARIO 1: User Requests Route Analysis
-----------------------------------------
User Input
  └─> Main.py triggers UrbanTrafficSystem class
      ├─> Auto-detect user location (LocationService + IP Geolocation)
      ├─> Get destination from user input
      ├─> Query Google Maps for routes
      ├─> Analyze traffic conditions (TrafficAnalyzer)
      ├─> Determine congestion level & delay
      ├─> Evaluate alternative routes
      ├─> Generate recommendations
      └─> Display results in formatted output


SCENARIO 2: Parking Violation Detection
----------------------------------------
ESP32 Sensor Detection
  └─> Vehicle too close to parking spot
      └─> ESP32 sends alert to Backend API
          └─> Backend validates data
              └─> Stores violation in Google Sheets
                  └─> Frontend displays alert
                      └─> Notification to enforcement team


SCENARIO 3: Traffic Statistics & Monitoring
--------------------------------------------
Dashboard View
  └─> Frontend requests /api/statistics
      └─> Backend processes Google Sheets data
          ├─> Calculate available parking spots
          ├─> Count violations
          ├─> Compute occupancy rates
          ├─> Aggregate system metrics
          └─> Return JSON response
              └─> Frontend renders visualizations

NECESSARY FUTURE DEVELOPMENTS & IMPROVEMENTS

PRIORITY 1: CORE FUNCTIONALITY ENHANCEMENTS
============================================

1. Database Migration
   Current: Google Sheets (limited scalability)
   Recommended: PostgreSQL / MongoDB
   Benefits:
   - Better performance for high-volume data
   - Complex query capabilities
   - Transaction support
   - Real-time data syncing
   Implementation: Add ORM (SQLAlchemy) with async support

2. Microservices Architecture
   Current: Monolithic structure
   Recommended: Split into microservices
   Services needed:
   - Authentication Service (JWT-based)
   - Traffic Analysis Service
   - Parking Management Service
   - Violation Detection Service
   - Notification Service
   Implementation: Docker containerization, Kubernetes orchestration

3. Real-time Data Streaming
   Current: Poll-based API calls
   Recommended: WebSocket/Server-Sent Events
   Benefits:
   - Live traffic updates
   - Instant alerts
   - Reduced latency
